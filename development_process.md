# 植物助手小程序 - 开发流程文档

## 项目概述
**植物助手**是一款基于 `uni-app` 框架开发的微信小程序。它旨在为用户提供一个集植物识别、养护知识科普、绿植商城以及个人植物管理于一体的综合平台。

---

## 技术栈
- **前端框架**: `uni-app` (支持多端发布，本项目主要针对微信小程序)
- **核心语法**: `Vue 3` (Composition API 风格)
- **样式处理**: `SCSS` (利用全局变量定义设计规范)
- **组件库**: `uni-ui` (如 `uni-icons`)
- **设计系统**: 自定义全局样式变量（见 `uni.scss`）

---

## 开发流程详解

### 第一阶段：环境搭建与项目初始化
1. **工具安装**: 安装 HBuilderX (或 VS Code + 插件) 以及微信开发者工具。
2. **项目创建**: 使用 `uni-app` 模板初始化项目，选择 Vue 3 版本。
3. **Git 初始化**: 建立版本控制，忽略 `unpackage/` 等生成目录。

### 第二阶段：全局配置与设计规范
1. **基础配色**: 在 `uni.scss` 中定义品牌色（如渐变绿 `$green-gradient`）、基础文字色、背景色等。
2. **路由规划**: 在 `pages.json` 中配置核心页面路径：
    - 首页 (`pages/index/index`)
    - 商城 (`pages/shop/shop`)
    - 知识库 (`pages/knowledge/knowledge`)
    - 个人中心 (`pages/profile/profile`)
3. **全局样式**: 设置 `globalStyle`，统一导航栏背景及文字色。由于本项目使用了大量自定义设计，多数页面开启了 `"navigationStyle": "custom"`。

### 第三阶段：核心通用组件开发
为了保证 UI 的一致性和高端感，本项目开发了以下关键组件：
- **自定义导航栏 (`custom-navbar`)**: 支持动态计算微信小程序“胶囊”位置，确保搜索框或标题不被遮挡。
- **自定义 TabBar (`custom-tabbar`)**: 替代原生 TabBar，实现更丰富的交互特效与品牌化设计。

### 第四阶段：功能模块实现

#### 1. 首页 (Home)
- **视觉区**: 实现品牌 Logo 与 Slogan 展示。
- **搜索与识别**: 集成快速搜索入口，并预留“花草识别”相机功能的触发逻辑。
- **快捷入口**: 采用 Grid 布局展示识别、指南、日历、医生等功能卡片。
- **今日推荐**: 响应式网格布局，展示热门养护指南。

#### 2. 商城 (Shop)
- **双栏交互**: 实现左侧侧边栏导航与右侧商品流的联动。
- **顶部分类**: 横向滚动的标签栏（Scroll-View），支持按需筛选。
- **商品卡片**: 包含品牌标签、规格说明、价格展示及一键加入购物车功能。

#### 3. 知识库 (Knowledge)
- **分类体系**: 纵向分类导航，覆盖养护技巧、病虫防治、品种介绍等多个维度。
- **文章列表**: 采用图文并茂的卡片流，展示阅读量、发布日期等元数据。

#### 4. 个人中心 (Profile)
- **用户档案**: 展示头像、昵称及等级信息。
- **统计面板**: 记录用户种植天数、收藏数量及浏览足迹。
- **功能列表**: 采用圆角卡片化设计，提供我的植物、系统设置等入口。

### 第五阶段：平台适配与优化
- **兼容性处理**: 利用 `uni.getSystemInfoSync()` 和 `uni.getMenuButtonBoundingClientRect()` 动态获取状态栏及胶囊高度，完美适配各种刘海屏手机。
- **性能优化**: 使用 `scroll-view` 的 `scroll-y` 和 `scroll-x` 实现局部滚动，避免页面整体拉伸引起的 UI 错位。
- **资源管理**: 静态资源（图标、背景图）统一存放于 `static/` 目录。

## 运行与发布

### 本地开发
1. 在 HBuilderX 中点击 `运行 -> 运行到小程序模拟器 -> 微信开发者工具`。
2. 确保微信开发者工具开通了“服务端口”权限。

### 发布上线
1. 在 HBuilderX 中点击 `发行 -> 小程序-微信`。
2. 生成的包位于 `unpackage/dist/build/mp-weixin` 下。
3. 在微信公众平台提交审核。
